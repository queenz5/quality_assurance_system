# 需求分析功能使用指南

## 📋 功能概述

需求分析功能允许你通过前端界面上传或粘贴需求文档，AI 会自动解析并生成结构化的需求数据。

## 🚀 使用步骤

### 步骤 1：访问需求分析页面

1. 启动前端：`cd frontend && npm run dev`
2. 启动后端：`cd qa_system && python main.py`
3. 访问：http://localhost:5173/requirement-analysis
4. 在左侧菜单点击 **"需求分析"**

### 步骤 2：输入需求内容

支持两种输入方式：

#### 方式 A：文件路径（需要文件已在服务器上）
- 输入服务器上需求文件的完整路径
- 例如：`/path/to/prd.md` 或 `/path/to/prd.pdf`
- 适用于文件已经上传到服务器的场景

#### 方式 B：文本输入（推荐）
- 在文本框中粘贴完整的需求文档内容
- 填写需求来源名称（可选）
- 适用于快速测试和少量需求

### 步骤 3：AI 解析预览

点击 **"开始解析"** 后，AI 会：
1. 识别功能模块
2. 提取功能点
3. 分析业务规则
4. 识别依赖关系

解析完成后显示预览表格：
- 需求 ID
- 需求标题
- 所属模块
- 优先级
- 功能点数

可以点击 👁️ 图标查看单个需求详情。

### 步骤 4：确认并保存

检查解析结果无误后：
1. 点击 **"确认并保存到 data 目录"**
2. 系统会为每个需求生成 Markdown 文件
3. 保存到 `qa_system/data/requirements/{模块}/` 目录
4. 显示保存成功的文件列表

### 步骤 5：查看结果

保存成功后可以：
- 点击 **"查看数据管理"** 跳转到数据管理页面
- 或点击 **"继续解析新需求"** 处理下一个需求

## 📂 输出格式

每个需求保存为独立的 Markdown 文件：

```
qa_system/data/requirements/
├── _index.json                    # 全局索引（自动更新）
├── 用户管理/
│   └── REQ-001_用户登录功能.md
└── 订单管理/
    └── REQ-002_订单创建功能.md
```

   - 约束条件: 待补充

## 后置条件
- 待补充

## 业务规则

## ⚠️ 注意事项

1. **当前版本限制**：
   - 文件路径模式需要文件已在服务器上
   - 文本输入模式更灵活，推荐优先使用

2. **内容长度**：
   - 后端限制输入 8000 字符
   - 超长文档建议分段解析

3. **人工审核**：
   - AI 解析后建议检查预览结果
   - 补充遗漏的功能点和业务规则
   - 调整优先级和依赖关系

4. **文件编码**：
   - 所有文件使用 UTF-8 编码
   - 支持中文内容

## 🔧 故障排除

### 问题：解析失败
- 检查 LLM 配置（`.env` 文件）
- 确认 API Key 有效
- 查看错误提示信息

### 问题：保存失败
- 检查后端服务是否运行
- 确认 `qa_system/data/requirements/` 目录有写权限
- 查看浏览器控制台和后端日志

### 问题：前端页面空白
- 确认前端服务已启动
- 检查浏览器控制台是否有错误
- 刷新页面重试
