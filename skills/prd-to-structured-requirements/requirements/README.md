# 需求文档目录

将需要生成结构化需求的**原始 PRD 文档**放在此目录下。

## 支持格式

- **PRD / 需求文档**：`.docx`（Word）、`.md`（Markdown）
- **PDF**：`.pdf`
- **纯文本**：`.txt`

## 生成的结构化需求格式

- Skill 会输出 **JSON** 格式的结构化需求数据
- 后续转换为 Markdown (.md) 文件保存到 `qa_system/data/requirements/` 目录
- 导入方式：通过 API 调用或前端界面上传

## 使用方式

在对话中说例如：

- 「解析这个需求文档，生成结构化需求」
- 「拆解 PRD 为结构化需求数据」
- 「读取 requirements 里的文件并生成结构化需求」

若未指定具体文件，Skill 会列出本目录下的文档供你选择后再生成。
